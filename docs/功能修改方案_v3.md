# 功能修改方案 v3（基于对抗评审裁决）

> 逐模块给出：【保留】【必改】【新增】【删/降级】。每条修改都对应评审已裁决的问题。
> 优先级：P0 = 不改会被答辩问穿 / 闭环走不通；P1 = 明显提升说服力；P2 = 锦上添花。
> 配套：[[项目选题对抗评审报告]] / [[开发文档写作蓝图_v3]]

---

## 一、三个全局改动（先定这三条，再改模块）

**G1【P0】核心场景收窄。** 主打**竞赛 / 大创 / 考研 / 就业 / 组队**这些高决策成本、强信息差场景；**生活圈**（校园网/快递/食堂/办事流程）降级为**只读引流入口**，高时效信息一律外链官方、不自存。
- 依据：评审证实生活信息是弱需求（群里问一句就解决），且时效性上你们结构性劣于官方和当届社群。把力气花在弱需求上是失分点。

**G2【P0】需求三层诚实声明。** 在需求文档里明确分层：
| 层 | 内容 | 系统态度 |
|---|---|---|
| 真需求 | 考研/竞赛/就业经验、校友真实路径、组队 | 核心覆盖 |
| 弱需求 | 生活信息 | 只做只读引流，不做核心 |
| 拿不到 | 导师避雷、保研内幕（需匿名） | 诚实声明不覆盖 |
- 依据：你们"实名无匿名"的纪律恰好排除了最高痛点却最敏感的信息。主动声明边界比假装全覆盖高分。

**G3【P0】设计全景 / 实现核心分离。** 文档里设计全部 7 模块，实现只做通核心闭环（M1+M3+M4+M7 审核），其余标注"设计完整、简化/未实现"。

## 二、逐模块功能修改

### M1 用户与认证
- **【保留】** 注册登录、角色权限、隐私设置。
- **【必改·P0】** "双身份认证"四个字 → **分级认证方案**：
  - 在校生：学号 + （可选）学校统一身份认证核验；
  - 毕业生：**学院/辅导员邀请码定向邀请**（机构侧确权、免本人举证），或"邮箱/学号 + 人工审核 + 同专业 2 人担保"。
- **【新增·P1】** 认证前只读 / 认证后可写的**权限分层**：生活圈公共 FAQ、校园网/快递等无隐私信息**认证前即可看**；发求助、填画像、看校友隐私字段才要求认证。
- **依据**：空洞认证认不住（毕业 3-10 年无权威源无法核验）；且认证卡住新生开学周急用信息会导致直接流失。

### M2 成长画像与校友路径
- **【保留】** 在校生画像/标签、毕业生路径卡、学长建议。
- **【必改·P1，升为 Must】** 新增**路径推荐**：输入（专业 / 年级 / GPA 区间 / 兴趣标签 / 目标城市行业）→ 规则引擎（按专业 + 意向标签过滤排序）→ 推荐发展路径与匹配校友。
  - 依据：原设计在最高价值的大三分流场景只有"看别人怎么走"（静态浏览+统计），相比知乎/贴吧没有增量，"明确成长方向"的立项理由落空。
- **【字段改·P1】** 路径卡加**"去向类型"分支字段**：选"深造"展开（目标院校/专业、初试成绩构成、复试经历、备考时长与资料）；选"就业"展开（城市/行业/公司/岗位）。
  - 依据：原字段只服务就业叙事，考研上岸细节无处安放，而考研是你们一等路线。
- **【加保护·P2】** 去向统计设**最低样本门槛**：某专业路径卡 < 20 条时显示"样本不足，仅供参考"并隐藏具体百分比。
  - 依据：小样本 + 幸存者偏差会系统性误导分流决策。
- **【字段加·P2】** 每个隐私字段的**可见性控制**（本人/同专业/公开）；去向**修订留痕**（版本）。

### M3 经验知识库
- **【保留】** 条目 CRUD、分类、搜索、时效/适用范围、三态评价、生命周期状态机。
- **【必改·P0】** 高时效信息（报到时间、选课截止、政策）**一律外链官方、绝不自存**；只自沉淀低时效经验（学习方法、备考经验、通用避坑）。
  - 依据："过期的权威信息比没有更危险"，用户踩一次雷就流失且负面口碑快。
- **【新增·P1】** 更新责任人机制：**"当届学生认领更新 + 内容过期自动降权"**，明确认领人角色。

### M4 结构化求助 ★最关键★
- **【保留】** 求助单、模板化回答、限次追问、采纳、生成知识候选。
- **【必改·P0，补上系统灵魂的缺环】** 新增 **"求助-校友路由匹配"**：求助单创建时按问题类型/目标方向/专业年级**自动匹配**对应标签的校友/学长，产生**站内通知或订阅推送**。验收标准：求助单发布后 N 小时内触发 ≥1 次匹配通知。
  - 依据：没有路由，求助单等于发到没人看的公告栏，采纳环节进不去，整条"求助→采纳→候选→审核→入库"闭环**空转**——这是全项目最致命的漏洞，直接决定"系统灵魂"成不成立。

### M5 机会与组队
- **【保留】** 机会状态机、组队/队伍状态机、**讲座作为类型枚举**（评审裁定保留是对的，复用同一状态机、边际成本近零）。
- **【必改·P1】** **内推从公共机会状态机拆出**。真实内推是"投递→内推→面试进度→结果"的私密个体追踪，公共状态机（一条信息面向所有人展示到截止）装不下。二选一：
  - 单独建模**"内推申请单"实体**（申请人/内推人/简历附件/私密进度：待处理→已内推→流程中→已结束）+ 独立可见性规则；
  - 或在报告中**诚实标注内推本期不做**（Could）。

### M6 成长时间线
- **【保留】** 按专业×年级×路线模板、节点只存关联 ID 零复制、个人进度标记。
- **【必改·P0，解自相矛盾】** 加**"未决策路线"默认通用时间线**。大一新生不该被迫先选考研/就业才能用时间线——那恰是"新生不迷茫"要解决的问题本身。默认时间线按大一上/大一下通用阶段给共性任务（选课/社团/四六级/分流准备）；四条分化路线（考研/就业/竞赛/考公）到大二下、大三上再作为进阶视图选择。
- **【新增·P1，升级为真导航】** 节点加**"当前时间 vs 建议时间"比对**，用户选定路线后自动算出已过期节点、生成**"补救优先级"提示**（已错过 X，现在最该做 Y）。
  - 依据：原来是静态模板，等于网上随手搜的规划图，"个性化成长导航"这个核心差异点没落地。

### M7 平台管理与内容治理
- **【保留】** 认证审核、知识候选审核、举报、机会/模板维护、标签维护、运营统计。
- **【必改·P0，让审核真能运转】**
  - **自动初审**：学号自动核验在校生身份，边缘案例才转人工；
  - **批量操作**：批量通过 / 批量退回 + 标准理由；
  - **知识候选"三秒可判断"隐私 checklist**：是否含真实姓名/联系方式/可反向定位组合，勾选即自动退回并附模板理由；
  - 定义**"高峰期最大日审核吞吐量"**非功能指标。
  - 依据：开学季认证 + 知识候选双队列指数堆积，10 分钟/天人工审核撑不住，会退化成"一键通过"或"不敢审"，直接击穿反贴吧可信度。

### 新增功能：贡献者激励（升为 Must，归入 M2/M7）
- **【新增·P1】** 贡献者认证标识 + **"已帮助 N 位学弟学妹"计数可见** + 被采纳次数可见 + 学院/团委承认的荣誉证明；每个需要校友结构化输入的入口**标注预计耗时**（"约 3 分钟"）。
  - 依据：决定内容供给量的激励机制原来排在 Could（可做可不做），把"系统怎么转"列 Must、"内容为什么会持续来"列最低，是本末倒置。

## 四、改完之后，"系统灵魂"闭环长这样（端到端）

发求助（带专业/年级/类型标签）
→ **路由匹配**：按标签自动通知对口校友 ★新增★
→ 校友模板化回答（适用前提/步骤/注意事项）
→ 提问者采纳最佳回答
→ 进入知识候选队列
→ 管理员审核（**自动初审 + 隐私 checklist**）★强化★
→ 入知识库（带时效/适用范围/来源）
→ 被**成长时间线节点**和**搜索**引用
→ 时间线按"当前进度 vs 建议时间"**动态推给**下一个需要它的人 ★新增★

这条链路是全项目最该演示、最该在文档里用状态图/时序图/DFD 反复展开的主线。原方案在第 2 步（路由）和最后一步（动态推送）是断的，补上后闭环才真正成立。
