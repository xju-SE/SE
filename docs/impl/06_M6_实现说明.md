# 06 M6 成长时间线 实现说明

> 代码位置：`backend/src/main/java/com/xju/sem/module/timeline/`
> 对应设计：`docs/design/06_M6_成长时间线_详细设计.md`；数据表以 `backend/src/main/resources/schema.sql` 的 `timeline_template / timeline_node / timeline_node_ref / user_progress` 为准。

---

## 1 模块功能说明

M6 把"专业 × 发展路线"的成长规划做成一份**持续对照真实时间、主动提醒**的动态成长导航，落地"看路径、查经验、问问题、找机会、跟时间线"里的"跟时间线"。实现的能力对应任务书 ①~⑧：

1. **时间线模板**（`timeline_template`，`major_tag_id × route_type`）：`route_type ∈ {UNDECIDED, POSTGRAD, EMPLOY, COMPETITION, CIVIL}`；`UNDECIDED` 为大一共性阶段的**通用默认线**，`major_tag_id = NULL` 表示全专业通用兜底模板。ADMIN 直接维护并自助发布（`DRAFT/PUBLISHED/OFFLINE`），不经 M7 审核队列。
2. **节点**（`timeline_node`）：`stage`（`GRADE1_1…GRADE4_2` 共 8 学期）、`suggested_time`（展示串）、`suggested_month`（机读月份，供逾期比对）、`importance`（1-3）、`order_no`。
3. **节点关联**（`timeline_node_ref`）：只存 `(ref_type, ref_id)` 引用 `ALUMNI_PATH_CARD / KNOWLEDGE_ENTRY / OPPORTUNITY`；**零复制**，展示时实时调各模块 `getVisiblePathCard / getBrief / existsPublished` 现取摘要。
4. **查看我的成长时间线**：未选路线且处于共性阶段走 `UNDECIDED` 默认线（首访懒初始化进度）；决策窗口后取最近确认的分化路线。
5. **选择/切换路线**：为目标模板全部节点批量 `INSERT IGNORE` 进度；`UNDECIDED` 不可主动选入。
6. **标记个人进度**（`user_progress`，`NOT_STARTED/DONE`）：`DONE` 写 `marked_at`，回切清空；越权防护校验节点属本人当前有效模板。
7. **★逾期动态判定 + 补救优先级**：按 `stage + suggested_month + enrollYear` 换算建议年月，与当前月做**月粒度**比对识别逾期节点；按"逾期分档 × `importance` + 学期邻近度"打分排序取 TopN 生成补救提示。
8. **对外契约** `TimelineNodeRefService.countByRef(refType, refId)`：供 M3 P09"该知识被 N 个成长时间线节点引用"等只读计数。

**边界（不做）**：不存画像/专业/入学年份（只读 M2）；不存被引用对象正文与状态机（只读 M2/M3/M5）；模板不走 M7 审核；`route_type` 是本模块封闭枚举，不与 M2 `destination_type`/M4 `tag(GROWTH)` 共用取值域；只覆盖本科 8 学期。

## 2 代码结构

```
module/timeline/
├─ controller/
│  ├─ TimelineTemplateController   (ADMIN /api/v1/timeline-templates：模板CRUD/publish/offline + 从属节点新增/列表)
│  ├─ TimelineNodeController       (ADMIN /api/v1/timeline-nodes：节点编辑/删除 + 引用GET/PUT)
│  ├─ TimelineStatsController      (ADMIN /api/v1/timeline-stats/by-major：专业级完成度)
│  └─ MyTimelineController         (STUDENT /api/v1/timeline/me… + /timeline-nodes/{id}/progress：聚合/预览/切换/进度/补救/统计)
├─ service/  (4 接口)  TimelineTemplateService / TimelineNodeService / TimelineNodeRefService / UserProgressService
│  └─ impl/
│     ├─ TimelineTemplateServiceImpl   (CRUD、§4.1 发布状态CAS、§6.2 解析+双场景查重、FR-12 统计)
│     ├─ TimelineNodeServiceImpl       (节点CRUD、字段范围校验)
│     ├─ TimelineNodeRefServiceImpl    (只存ID+现取展示+覆盖式维护、§6.7 countByRef)
│     ├─ UserProgressServiceImpl       (核心：§6.3 路线解析/切换、§6.4 逾期、§6.5 补救、§6.6 懒初始化)
│     └─ TimelineMapping               (实体→DTO 与节点排序，包内复用)
├─ util/  TimelineCalendarUtil         (§6.1 学期日历换算纯函数，无DB依赖、可单测)
├─ entity/  TimelineTemplate / TimelineNode / TimelineNodeRef / UserProgress
├─ mapper/  TimelineTemplateMapper / TimelineNodeMapper / TimelineNodeRefMapper / UserProgressMapper
├─ enums/   RouteType / TemplateStatus / Stage / RefType / ProgressStatus / TimelineErrorCode
└─ dto/  request(9) + response(15)
```

- `TimelineTemplate`/`TimelineNode` 继承 `BaseEntity`（含逻辑删除与审计时间）；**`TimelineNodeRef`/`UserProgress` 不继承 `BaseEntity`**——schema 这两张表无 `deleted/created_at/updated_at` 列，故引用走物理"删旧插新"、进度走 `INSERT … ON DUPLICATE KEY UPDATE`。
- Controller 只做参数校验转发，返回 `Result<T>`/`Result<PageResult<T>>`；REST 前缀 `/api/v1`，状态流转用 `PATCH`。

### 对外/对内契约
- **对外提供**（供他模块）：`TimelineNodeRefService.countByRef(String refType, Long refId): long`（供 M3 P09 计数，签名与地基契约一致）。
- **对内提供**：`TimelineTemplateService.resolve(Long majorTagId, String routeType): TimelineTemplateDTO`（§6.2 解析，含通用兜底），供 `UserProgressServiceImpl` 复用。
- **依赖他模块**（只调 Service 接口，绝不直连其 Mapper）：`StudentProfileService.getProfile`（取 `majorTagId/enrollYear`）、`AlumniPathCardService.getVisiblePathCard/existsPathCard`、`KnowledgeEntryService.getBrief/existsPublished`、`OpportunityService.getBrief`。

## 4 关键实现点

1. **schema 优先、对详细设计的裁剪对齐**（均以 `schema.sql` 为准）：
   - 逾期比对：设计稿以"周次 + 起始日精确到天"，schema 落地为 `suggested_time VARCHAR`（展示串）+ `suggested_month TINYINT`（机读月份）。实现改为 `TimelineCalendarUtil` 以 **stage + suggested_month + enrollYear** 换算"建议自然年月"，按**月粒度**判逾期（对齐任务书"suggested_month vs 当前月"），同时给出 `daysOverdue`（= 建议月月末到今日）供展示层选用，规避 schema 未存周次的伪精度。
   - `importance`：schema 为 TINYINT 1-3（越大越关键），补救打分权重直接取该值（替代设计稿 KEY/RECOMMENDED 的 1.5/1.0）。
   - `timeline_template` 无 `description` 列、无唯一索引：模板简介不落库；NULL 通用模板与非 NULL 专业模板的**查重全部在 Service 应用层兜底**（`countGeneric`/`countByMajorRoute`）。
   - `timeline_node_ref` 无 `ref_order`：展示顺序取请求/插入自然序；`user_progress` 用 `marked_at`（非 `done_at`）。
2. **学期日历换算（`TimelineCalendarUtil`，纯函数可单测）**：`Stage` 携带 `order/是否秋季/yearOffset` 元数据；秋季学期跨自然年（8-12 月归学期起始年，1-7 月溢出到次年）；`currentStage` 反查当前学期，早于大一上钳位、晚于大四下+7 月判毕业返回 null。
3. **路线解析（§6.3，不建第 5 张表）**：共性阶段恒用 `UNDECIDED`；决策窗口后 `findConfirmedRoute` 用 `user_progress JOIN node JOIN template` 取"最近一条非 UNDECIDED"路线（schema 无 `created_at`，以自增 `up.id DESC` 代插入序）。通用线与分化线学期区间不重叠，保证反查唯一。切换路线保留旧进度为历史，仅不再被"最近一条"选中。
4. **零复制引用现取（§6.7）**：`timeline_node_ref` 只存 `(ref_type, ref_id)`；`replaceRefs` 覆盖式重建前逐条按 `refType` 调对应模块 `existsPathCard/existsPublished/getBrief` 校验存在性（不存在 → 40601）；展示 `resolve` 对每条现取摘要并 `try/catch` 兜底，任一失效只标 `available=false`，不拖垮聚合视图。`countByRef` 为对外只读计数契约。
5. **并发与事务**：模板发布态走状态 CAS（`UPDATE … WHERE id=? AND status=期望前置态`，受影响 0 行即冲突，schema 无 version）；`user_progress` 因"同一行只本人一个写者"用 `INSERT … ON DUPLICATE KEY UPDATE`（upsert）与 `INSERT IGNORE`（批量初始化保留原进度），不需乐观锁；`confirmRoute` 批量初始化用 `@Transactional`。
6. **越权防护**：`markProgress` 在 Service 层校验目标节点 `template_id` 等于该用户 `resolveEffective` 解出的有效模板 id，否则 40601，防止越权标记他人/非当前生效路线的节点。
7. **错误码**：模块内 `TimelineErrorCode` 以 `int` 常量 + `BusinessException(int,message)` 抛出（20601/20602/30601-30604/40601…），不侵入地基 `ResultCode` 枚举；分段对齐地基（2xxxx 参数、3xxxx 业务、4xxxx 资源）。
8. **首页/统计的容错**：`getMySummaryCard`/`getRemediationHints` 对模板未配置等 `BusinessException` 做降级（返回空卡/空列表），保证首页仪表盘不因单卡失败整页报错；`getMajorTimelineStats` 未配置模板时返回全零而非报错。
9. **未接入项（本期范围外，留扩展位）**：站内通知（"仍未决策临界提醒""补救预警"）本模块设计为触发方之一，但**本期不设强制定时任务**（逾期/补救均请求时实时计算），故未接 `NotificationService.send`；如需可在决策窗口检测 `needsRouteDecision` 或首页拉取时补发一条 `SYSTEM` 通知。ADMIN 维护端点已在 M6 自身路由提供（`/api/v1/timeline-templates`、`/api/v1/timeline-nodes`）；若 M7 另建 `/api/v1/admin/timeline-*` 别名，直接复用本模块 Service Bean 即可，不另建 schema。

### 假设与简化
- 假设 STUDENT 调用 `/timeline/me` 系列前其 `student_profile` 已存在且 `enroll_year` 非空（认证后完善画像的既有流程保证）；缺 `enroll_year` 时返回 30601 提示。
- 假设 `major_tag_id` 合法性由 ADMIN 录入保证（本期未接 `TagQueryService` 做存在性校验，避免对尚未落地的 M7 只读标签服务产生编译期强依赖）；接入后可在模板创建处补一次只读校验。
- 节点关联引用在聚合视图中逐节点现取，属"最佳努力"展示；数据量增大后可批量预取或按节点懒加载优化，本期按项目规模直接现取。
