# XJourney·疆行 论文写作套件（对齐真实代码 · 面向对象 · 正式论文口径）

本套件基于项目真实代码（后端 379 文件/2万行、前端 67 文件/近8千行、29 表、9 模块）编写，供团队喂给 ChatGPT 扩写到约130页。

## 六份文档
1. **01_论文写作指导与放图-第1至5章.md** — 引言/可行性/开发计划/需求分析/概要设计：每节 写什么·怎么写·可引用素材·放哪些图表·扩写提示。
2. **02_论文写作指导与放图-第6至9章.md** — 详细设计(逐模块)/测试/操作手册/总结：含模块-代码对应表、函数设计表、测试用例表等模板格式。
3. **03_新增图画图提示词-16张.md** — 需新画的 16 张 UML/E-R/架构图，每张含元素/关系/Visio模具/可复制画图提示词。
4. **04_已有图修改提示词.md** — all文件夹已画图的修改说明（含图6-4虚构图删除、图5-2改包图、loginByRole→login等）。
5. **05_截图清单与测试清单.md** — 逐条界面截图 + 测试用例（可直接执行）。
6. **06_项目亮点与安全性写作素材.md** — 自研UIKit/背景、AFTER_COMMIT事件链、路由算法、JWT/BCrypt/越权防护等，含写进哪节。
7. **07_表格清单与放置指南.md** — 全书表格总览 + 逐表(放哪/表头/内容),含版本迭代记录表(git)、责任矩阵、权限矩阵、路由权重表、状态迁移表、徽章表、UIKit组件表、缺陷与修复记录表(对应模板测试章缺陷/建议)。

## 三条通用约束（全程遵守）
- **面向对象方法学**：系统建模只用 UML（用例/类/时序/活动/状态/包图）+ E-R图；不用任何数据流图(DFD)/结构化流程图/结构图。
- **正式论文口径**：通篇不出现“演示模式/真实模式/演示态/未落库/假数据/未实现”等字样，以一个完整系统撰写。
- **对齐真实代码**：类名/表名/字段/状态/接口/数字一律以真实项目为准，扩写时不臆造新事实。
